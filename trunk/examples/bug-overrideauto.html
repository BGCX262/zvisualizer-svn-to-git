IE6 / IE7

Bug where the z-index cannot be reset if it is given a non-zero starting value.
In this example, #div7 (z-index:7) appears above #div1 (z-index:7)
Setting the z-index to 'auto' or 0 has no effect.

  • Change the z-index to "auto" (supposed to send it to the back)
  • Change the z-index to 0 (supposed to send it to the back)
  • Change the z-index back to the original
