Firefox3+ / Safari3+

Non-full opacity will implicitly create a stacking context.
#div3 has no positioning (and no z-index), but has opacity, so the children of #div3 will be inside #div3's stacking context.

Firefox2

For elements with opacity but no positioning, how they are sorted in relation to other positioned elements is different in firefox2, when compared to firefox3+/safari3+. From my testing, it appears that firefox2 groups all of these above the float layer, but under the positioned layer. In firefox3+/safari3+ they are sorted within the positioned layer.

IE6/7/8

Note that in IE, opacity can only be applied to positioned elements. So if you wanted opacity, you would have to apply positioning, which automatically gives it stacking context in IE.
